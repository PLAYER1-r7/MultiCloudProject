## Summary
MVP でも失敗時に安全に戻せることが必要であり、アプリとインフラの切り戻し方法を事前に定める必要がある。

## Goal
配信物、インフラ、DNS、認証設定の切り戻し方針を定義する。

## Tasks
- [ ] アプリ配信物のロールバック方針を定義する
- [ ] インフラ変更のロールバック方針を定義する
- [ ] DNS 変更のロールバック方針を定義する
- [ ] 認証設定のロールバック方針を定義する
- [ ] 復旧確認方法を定義する
- [ ] 想定復旧時間を整理する
- [ ] ロールバック手順書を作成する

## Definition of Done
- [ ] アプリ配信物を前回の正常版へ戻す手順と単位が整理されている
- [ ] IaC を前提にしたインフラ切り戻し方針と例外時の扱いが説明されている
- [ ] DNS または配信経路の変更を戻す判断手順が整理されている
- [ ] 運用アクセスや認証まわりで切り戻し対象となる範囲が明示されている
- [ ] ロールバック後に実施する最小復旧確認フローが説明されている
- [ ] 初回リリースにおける実用的な想定復旧時間の目線が整理されている
- [ ] インシデント時の復旧判断材料として参照できる状態になっている
